在github搭建博客的官方指引链接

https://pages.github.com/

整篇文章中只有一个要遵从的地方，就是建立的仓库名称必须为：username.github.io。后面的步骤不需要，只需要一个空的仓库。

如何修改域名

修改域名需要付费，懒得弄。参考链接
https://help.github.com/articles/using-a-custom-domain-with-github-pages/

使用hexo

hexo是github上一个开源项目，基于node.js，帮助快速搭建博客，可以支持markdown。官网链接：https://hexo.io/

使用hexo的简单步骤

安装

  1. 安装git bash。没啥可说，肯定装了，而且用的很6666。
  2. 安装node.js。直接百度下载最新版本。
  3. 安装hexo。打开git bash，输入 npm install -g hexo-cli 。npm 是JavaScript的包管理工具，服务器在国外，有时候会比较慢，install的时候要有耐心，或者用代理。实在慢的不行，可以百度关键词 “npm加速”。
  4. 安装hexo-deployer-git。打开git bash ，输入 npm install hexo-deployer-git，要有耐心。
  5. 安装hexo-server。打开git bash ，输入 npm install hexo-server –save，要有耐心。

初始化和配置

  1. 在硬盘中选一个你心仪的位置，然后在git bash中输入 hexo init [folder] 。然后你会看到folder中一堆文件。
  2. hexo的配置都在folder中名字为 _config.yml 的文件中，至于怎么配置，请参考hexo官网。

生成和部署

  1. hexo folder的文件夹里面有一个路径source_posts，在其中放入 xxx.md，就完成内容更新了。当然也可以在git bash中输入 hexo new [layout] 它会帮你生成一个markdown文件。
  2. 在git bash 中输入 hexo generate 即可生成。
  3. 在部署到github上之前，可先在本地查看一下。在git bash中输入 hexo server，稍等一会就在本地打开了一个4000端口，只需在浏览器中输入 http://localhost:4000/ 即可访问。
    不过有时候这个端口被占用，就会出现一直打不开的情况(git bash中还是显示打开了)。这个时候需要换一个端口试一下，输入 hexo server -p 5000 即可换到5000端口。
  4. 觉得没问题了，输入 hexo deploy 即可。然后在浏览器输入 username.github.io ，就可以欣赏你的杰作了。

图床问题

markdown外链图片要放在服务器上。国内有很多免费的图床，但是都不敢太信任，怕某一天他们就把服务器给撤了。github本身就有仓库，不用白不用(当然这有违github的宗旨，所以不赞成。)
使用方法很简单。

  1. 建立一个仓库。
  2. 把图片push上去。
  3. 点击仓库中图片，得到链接。不过这个时候，外链是打不开的。比如我的一个仓库中的图片链接https://github.com/weynechen/LinkOutPicture/blob/master/GraphVizNote/graph.png

直接markdown外链这个图片
会发现没有成功。

这时只需要将链接中的blob改成raw就可以了。

再次外链：

不过，hexo提供了一个 assert folder 机制。当使用 hex new filename 生成markdown文档时，会同时生成一个filename文件夹，将图片放在其中，他会一并上传github，就不需要图床来外链了。详情参考
https://hexo.io/docs/asset-folders.html

更换hexo 主题。

hexo可以更换主题，并且主题还很多。可以参考知乎问题：有哪些好看的 Hexo 主题？，另外hexo官网也有。

更换步骤：

  1. 在git上找到主题
  2. 切换到themes目录
  3. git clone 主题
  4. 修改hexo的_config.yml文件。将theme的参数改成该主题名字

更多的请参考hexo官网。
