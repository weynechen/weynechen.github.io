总线速率

  • usb3.0支持超高速(5Gbps)
  • usb2.0支持高速(HS，480Mbps)，全速(FS,12Mbps)，低速(LS,1.5Mbps)
  • 速率指的是总线速率(bus speed)，并不是用户数据传输的速率。因为一个传输协议里面，除了用户自己的数据，还可能有控制数据，错误校验数据等。而且可能存在多个设备共用一条总线的情况。

有效数据速率

单一设备，超高速约400Mbps，高速约53Mbps，全速约1.2Mbps，低速约800bps

几个术语

  1. 设备 – 主机

    和 I2C 通讯的主从(master/slave)一样，USB叫host/device。在通讯里面，都是由主的一方先发起。

  2. 端口(port)

    目前还不太理解，相对于设备的endpoint，应该是主机端给设备分配的一个缓冲区。

  3. 端点(endpoint)

    • 设备的缓冲区，用来缓存接收到的数据和待发送的数据。STM32支持的端点数需要查询手册。
    • 端点分为IN端点和OUT端点（从主机方向看）。
    • 每个设备都必须有端点0，主机才能使用控制传输完成枚举功能。

  4. 事务(transaction)

    理解为一次传输所传递的信息。这些信息表明这次传输用来干什么，发了什么数据等等。
    事务可能有三个步骤：

    • token packet（令牌包）
    • data packet （数据包）
    • handshake packet （联络包)

  5. 管道(pipe)

    管道是主机软件和设备端点之间的联系，是主机枚举完设备后建立的。这个应该是抽象层次比较高的通信了。一个USB设备可能和主机之间有多个管道。

  6. 令牌(token)

    暂时不管。

传输类型

  1. 控制传输

    必定支持的传输类型，枚举时用来读取设备的信息。

  2. 批量传输

    数据量大，但是对实时性不太重要的场合使用，比如U盘。只管发送，中间有点速率变化无所谓。

  3. 中断传输

    中断的要求是及时响应，比如鼠标，键盘。

  4. 同步(等时)传输

    对发送的时间间隔要求比较严格，比如音频，视频。如果出现总线忙，声音就会不连续了。

枚举(enumeration)过程

主机获取设备信息，并为设备分配驱动程序的过程称为枚举。

  1. power state

    集线器检测到有设备接入，提供100mA的容许电流

  2. 集线器检测到设备连接

    集线器检测D+ 或者D-是否被拉高来检测是否有设备接入。

  3. 集线器报告端口信息

    报告是hub发出一个中断，主机发送一个Get port status请求，hub响应请求

  4. 集线器检测设备是低速还是高速，并报告给主机

    • 上拉电阻在D+，是高速设备
    • 上拉电阻在D-，是低速设备

  5. 集线器复位设备

    将D+,D-拉低10ms以上即完成复位。

  6. 了解设备是否支持高速。

    这一过程是通过两个特殊信号完成。暂时不管

  7. 主机得知设备复位完成，准备开始用端点0控制传输。

  8. 主机发送Get Descriptor，获取设备maximum packet size

  9. 主机为设备指定一个地址。

  10. 主机发送Get Descriptor,获取设备描述字。

  11. 主机加载驱动程序

枚举(enumeration)注意事项

设备收到主机的setup 事务时，必须放弃当前传输而开始新的传输。如果在枚举成功前，设备一直在发送数据，就有可能无法及时响应主机的setup事务，从而导致枚举失败，这是用户程序编写不当导致的枚举失败
